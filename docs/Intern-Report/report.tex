\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}
This report details the design and development of \textbf{VerbaSense}, a high-performance \Gls{stt} application developed during my internship at \Gls{ussbd}. VerbaSense leverages the OpenAI Whisper engine to provide secure, local, and accurate audio transcriptions. 

The project was built using a modern full-stack architecture, including Next.js 14 for the frontend and Node.js/Express for the backend. Key features include real-time transcription, history management, and secure \Gls{jwt}-based authentication. This report covers the methodology, system architecture, implementation details, and evaluation of the platform.

\printchaptertitle{Technical Projects and Assignments}
\section{Internship Goals and Deliverables}
The main goal was the completion of the VerbaSense \Gls{stt} platform, ensuring it was ready for local deployment and testing.

\section{VerbaSense: System Architecture}
VerbaSense follows a decoupled client-server architecture. The frontend communicates with the backend via a REST \Gls{api}, and the backend interacts with the Whisper engine and MongoDB database.

\section{Technology Stack Implementation}
\begin{itemize}
    \item \textbf{Frontend:} Next.js 14 (App Router).
    \item \textbf{Backend:} Node.js/Express.js.
    \item \textbf{Engine:} OpenAI Whisper.
    \item \textbf{Database:} MongoDB.
\end{itemize}

\section{Development Workflow and FFmpeg Integration}
We utilized FFmpeg for automated audio format normalization, ensuring that all user-uploaded files were converted to a standard format compatible with the Whisper engine.

% --- Chapter Four ---
\printchaptertitle{Technical Challenges and Solutions}
\section{Audio Processing Hurdles}
One of the main challenges was handling large audio files and ensuring low-latency transcription. This was solved by optimizing the Whisper C++ CLI implementation and utilizing efficient file streaming.

\section{Security and JWT Implementation}
Implementing a secure authentication layer required careful handling of tokens and user sessions. We utilized \Gls{jwt} for stateless authentication, ensuring that transcription history remains private to each user.
