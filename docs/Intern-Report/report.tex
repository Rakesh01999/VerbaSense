\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}
This report details the design and development of \textbf{VerbaSense}, a high-performance \Gls{stt} application developed during my internship at \Gls{ussbd}. VerbaSense leverages the OpenAI Whisper engine to provide secure, local, and accurate audio transcriptions. 

The project was built using a modern full-stack architecture, including Next.js 14 for the frontend and Node.js/Express for the backend. Key features include real-time transcription, history management, and secure \Gls{jwt}-based authentication. This report covers the methodology, system architecture, implementation details, and evaluation of the platform.

\printchaptertitle{Introduction}
\newpage

\section{Background}
In the modern digital era, the demand for accurate and efficient transcription services has increased significantly across various industries. VerbaSense was conceptualized to address the need for a secure, localized \Gls{stt} solution that minimizes dependency on expensive cloud \Gls{api}s while maintaining high transcription quality \cite{whisper}.

\section{Objectives}
The primary objectives of the VerbaSense project are:
\begin{itemize}
    \item To develop a robust web application for audio transcription using Next.js \cite{nextjs}.
    \item To integrate the Whisper engine for local speech processing.
    \item To ensure data security and user privacy through local hosting and \Gls{jwt} authentication.
    \item To provide a premium user experience with modern UI/UX design.
\end{itemize}

\section{Scope}
The scope of this project includes the development of a React-based frontend, a secure RESTful \Gls{api} using Express \cite{express}, database management for transcription history using MongoDB \cite{mongodb}, and the optimization of audio processing pipelines using FFmpeg \cite{ffmpeg}.

\printchaptertitle{Methodology}

\section{System Architecture}
VerbaSense follows a decoupled client-server architecture. The frontend communicates with the backend via a REST \Gls{api}, and the backend interacts with the Whisper engine and MongoDB database.

\section{Technology Stack}
\begin{itemize}
    \item \textbf{Frontend:} Next.js 14 (App Router), Tailwind CSS, Framer Motion.
    \item \textbf{Backend:} Node.js, Express.js.
    \item \textbf{Transcription Engine:} OpenAI Whisper (C++ CLI Implementation).
    \item \textbf{Audio Processing:} FFmpeg.
    \item \textbf{Database:} MongoDB (Mongoose ODM).
    \item \textbf{Authentication:} JSON Web Tokens (\Gls{jwt}).
\end{itemize}

\section{Development Workflow}
The project followed an Agile methodology, with iterative development cycles. We used Git for version control and focused on modular code structure to ensure scalability and maintainability.

\printchaptertitle{Implementation}

\section{Core Features}
\begin{itemize}
    \item \textbf{Secure Transcription:} High-accuracy conversion using the Whisper medium model.
    \item \textbf{Interactive Dashboard:} Real-time status updates and visualizations.
    \item \textbf{History Management:} Users can view, search, and delete past transcriptions.
    \item \textbf{Format Normalization:} Automated conversion of various audio formats (WAV, MP3, FLAC) using FFmpeg.
\end{itemize}

\section{UI/UX Design}
The user interface was built with a ``premium'' aesthetic in mind, utilizing Shadcn/ui components and smooth animations provided by Framer Motion. The layout is fully responsive, ensuring a seamless experience across devices.

\printchaptertitle{Results and Discussion}

\section{Performance Evaluation}
VerbaSense demonstrated significant efficiency in processing audio files locally. The use of the Whisper C++ implementation allowed for near real-time transcription on standard server hardware without the need for external cloud costs.

\section{Transcription Accuracy}
Testing against various accents and audio qualities showed that the Whisper medium model provides an Word Error Rate (WER) comparable to industry leaders, particularly for clear English and technical terminology.

\printchaptertitle{Conclusion}

\section{Summary}
The development of VerbaSense successfully met the objective of creating a professional, secure, and accurate Speech-to-Text platform. The integration of cutting-edge technologies like Whisper and Next.js 14 ensured a high-quality product suitable for enterprise use.

\section{Recommendations}
For future enhancements, we recommend:
\begin{itemize}
    \item Implementing multi-language support (Whisper supports 99 languages).
    \item Adding real-time streaming transcription via WebSockets.
    \item Enhancing the speaker diarization feature to identify multiple speakers.
\end{itemize}
