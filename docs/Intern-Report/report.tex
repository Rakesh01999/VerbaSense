\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}
This report details the design and development of \textbf{VerbaSense}, a high-performance \Gls{stt} application developed during my internship at \Gls{ussbd}. VerbaSense leverages the OpenAI Whisper engine to provide secure, local, and accurate audio transcriptions. 

The project was built using a modern full-stack architecture, including Next.js 14 for the frontend and Node.js/Express for the backend. Key features include real-time transcription, history management, and secure \Gls{jwt}-based authentication. This report covers the methodology, system architecture, implementation details, and evaluation of the platform.

\printchaptertitle{Introduction}
In the rapidly evolving landscape of Software Engineering and Artificial Intelligence (AI), theoretical mastery must be complemented by rigorous practical application to develop industry-standard solutions. As global industries undergo a massive digital shift, academic institutions are prioritizing internships to bridge the gap between classroom algorithms and production-ready deployments. For a Computer Science and Engineering student, an internship at a specialized software firm offers an unparalleled opportunity to understand the complexities of the Software Development Life Cycle (SDLC) and the nuances of integrating advanced AI engines into user-centric platforms.

This report provides a detailed reflection on my professional journey as a Software Engineering Intern at \textbf{Unicorn Software \& Solutions Ltd (USSBD)}. Since its inception in 2005, \Gls{ussbd} has been at the forefront of the local IT sector, specializing in custom enterprise applications and innovative web architectures. My tenure at the Khulna branch was centered on the development of \textbf{VerbaSense}---a high-performance Speech-to-Text (\Gls{stt}) application designed to bring the power of the OpenAI Whisper engine to a secure, locally-hosted environment.

The primary objective of this report is to analyze the technical architecture of VerbaSense, document the development methodologies employed, and evaluate the performance of the integrated AI models. Furthermore, it highlights the professional growth and technical proficiency gained during this tenure, providing a technical roadmap for building scalable AI-integrated web applications.

\section{Overview of the Internship Program}
The internship program at \textbf{USSBD} is meticulously structured to offer final-year undergraduate students a holistic and immersive introduction to the professional world of software engineering, systems architecture, and large-scale project deployment. My journey at the Khulna branch was not merely a passive observation period; it was a carefully segmented 12-week program designed to bridge the gap between academic theory at Jashore University of Science and Technology (JUST) and the rigorous demands of the global IT industry. From the very first week, the program emphasized a "learning by doing" philosophy, where I was integrated into the core development team and given real responsibilities under the supervision of senior developers.

Unlike many standard internships that often restrict students to minor administrative tasks, USSBD’s framework focuses on engaged participation and professional mentorship. I was encouraged to rotate through different phases of the development lifecycle—from initial requirements gathering and UI/UX prototyping to backend API design and AI engine optimization. This rotation ensured a well-rounded experience, allowing me to understand how individual components of a complex application like VerbaSense come together to form a seamless user experience. The mentorship I received included regular technical briefings, deep-dive code reviews, and daily stand-up meetings that simulated a high-velocity agile environment.

\section{Specific Internship Modules}
To ensure structured technical growth and a deep understanding of modern web architectures, the internship was divided into two key technical modules, each focusing on a distinct layer of the VerbaSense ecosystem:

{\large\textbf{Module 1: Full-Stack Web Development (Frontend \& Backend)}}

\textbf{Location:} Khulna Branch Office, Nirala R/A
\begin{itemize}
    \item \textbf{Duration:} Approximately 7 weeks
    \item \textbf{Objective:} Master the end-to-end development of modern, scalable web applications using the Next.js and Express ecosystems while adhering to industry-standard security and performance benchmarks.
    \item \textbf{Key Learning Tools:} Next.js 14 (App Router), Node.js, Express.js, MongoDB, Tailwind CSS, Postman, and GitHub.
    \item \textbf{Learning Activities:}
    \begin{itemize}
        \item \textbf{Responsive UI Design:} Designing and implementing fluid, mobile-first user interfaces using Tailwind CSS and Next.js Server Components to ensure optimal load times and SEO performance.
        \item \textbf{API Architecture:} Developing robust RESTful API endpoints for handling secure user authentication (JWT), file upload streams, and real-time state management on the backend.
        \item \textbf{Database Management:} Implementing and optimizing MongoDB schemas for the persistent storage of complex transcription metadata and user history, focusing on data integrity and query efficiency.
        \item \textbf{Testing and Debugging:} Performing rigorous end-to-end testing of the application flow using tools like Postman to identify and resolve performance bottlenecks in the API layer.
    \end{itemize}
\end{itemize}

{\large\textbf{Module 2: AI Engine Integration and Audio Processing}}

\textbf{Location:} Khulna Branch Office, Nirala R/A
\begin{itemize}
    \item \textbf{Duration:} Approximately 5 weeks
    \item \textbf{Objective:} Gain deep technical expertise in the deployment of locally-hosted AI models and the complexities of handling high-performance audio processing in a web environment.
    \item \textbf{Key Learning Tools:} OpenAI Whisper (C++ CLI), FFmpeg, Node.js Child Processes, and system-level resource monitoring tools.
    \item \textbf{Learning Activities:}
    \begin{itemize}
        \item \textbf{AI Model Deployment:} Researching, deploying, and fine-tuning the OpenAI Whisper engine for local inference, ensuring the system can handle transcription tasks without relying on external cloud APIs.
        \item \textbf{Audio Pre-processing:} Developing automated scripts using FFmpeg to normalize, trim, and convert diverse audio formats (MP3, WAV, FLAC) into optimized streams for the transcription engine.
        \item \textbf{Resource Optimization:} Analyzing and optimizing the CPU and RAM allocation for concurrent AI inference tasks to ensure the server remains responsive during heavy transcription loads.
        \item \textbf{Secure Integration:} Designing stateless authentication layers and secure file-handling protocols to ensure that user-uploaded audio remains private and secure during the transcription process.
    \end{itemize}
\end{itemize}

\printchaptertitle{Technical Challenges and Solutions}
\section{Challenges Faced and Solutions}
\subsection{Large File Upload Handling}
Solved by implementing chunked uploads and increasing the payload limit on the Express server.
\subsection{Model Inference Latency}
Addressed by using the 'tiny' and 'base' models for quick drafts while reserving 'large' for final transcripts.
\subsection{Audio Format Incompatibility}
Handled through automated FFmpeg scripts that sanitize incoming audio streams.
\subsection{Secure Session Management}
Implemented through \Gls{jwt} and HTTP-only cookies to prevent unauthorized access.
\subsection{Resource Optimization for Local Deployment}
Optimized CPU usage by limiting the number of threads used by the Whisper engine during peak loads.
